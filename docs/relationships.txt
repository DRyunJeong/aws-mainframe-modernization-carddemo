 표기: A → B  (A가 B를 참조/호출/사용)

 [5] JCL 선행/후행 체인 (야간 배치 순서)
================================================================================
  근거: 루트 README "Running Batch Jobs" + scripts/run_full_batch.sh
        + 각 JCL 문서의 선행/후행 기술 + 메모리 [batch_and_optional_modules]

--- 핵심 야간 사이클 (CLOSEFIL 시작 → OPENFIL 끝) ---
CLOSEFIL
   → (마스터 갱신) ACCTFILE, CARDFILE, XREFFILE, CUSTFILE, TRANBKP/TRANFILE,
                  DISCGRP, TCATBALF, TRANTYPE, DUSRSECJ   [상호 독립, 병렬 가능]
   → POSTTRAN  (CBTRN02C: DALYTRAN 검증·전기 → TRANSACT KSDS 기록, 잔액/카테고리 갱신)
   → INTCALC   (CBACT04C: 이자 계산 → SYSTRAN(+1) 이자거래 생성, ACCTDATA REWRITE)
   → TRANBKP   (TRANSACT.VSAM.KSDS → TRANSACT.BKUP(+1) 백업 후 KSDS 재초기화)
   → COMBTRAN  (SORT: BKUP(0) + SYSTRAN(0) 병합 → COMBINED(+1) → KSDS 재적재)
   → TRANIDX   (IDCAMS: 재적재된 KSDS에 거래 AIX 재정의/빌드)
   → OPENFIL   (SDSF: CICS에 VSAM 전체 재오픈 → 온라인 거래 재개)

  ※ run_full_batch.sh 실제 호출 순서(주의: 스크립트는 데이터 리프레시 단계에서
    "Refresh Transaction Data"에 TRANBKP.jcl을 사용하고, 사이클 후반 백업에도
    TRANBKP.jcl을 재사용한다 — 동일 잡이 시퀀스에 두 번 등장):
      CLOSEFIL → ACCTFILE → CARDFILE → XREFFILE → CUSTFILE → TRANBKP → DISCGRP
      → TCATBALF → TRANTYPE → DUSRSECJ → POSTTRAN → INTCALC → TRANBKP
      → COMBTRAN → TRANIDX → OPENFIL

--- GDG/리포트 정의 잡 (사이클 전 1회성 선행) ---
DEFGDGB → (TRANSACT.BKUP/DALY/COMBINED, SYSTRAN, TCATBALF.BKUP, TRANREPT 베이스 정의)
            ↳ 후행: TRANBKP, COMBTRAN, INTCALC, TRANREPT 등이 (+1)세대 생성/소비
DEFGDGD → (TRANTYPE/TRANCATG/DISCGRP 참조데이터 GDG 정의+최초세대)
